# Development Rules (Web)

## 기본 원칙
- UI, 상태, API 모듈 책임을 분리한다.
- API DTO 타입을 정의하고 `any` 사용을 피한다.
- 실패를 조용히 무시하지 않는다.

## 코드 규칙
- 함수형 컴포넌트 + 훅 기반으로 구현한다.
- API 모듈은 기능 단위로 분리한다.
- 공통 상수(URL, 타임아웃, 딜레이)는 중앙 관리한다.
- 신규 기능은 최소 1개 이상의 검증 코드(테스트 또는 스모크 시나리오)를 추가한다.

## 상태/동기화
- 낙관적 업데이트 시 롤백 경로를 반드시 구현한다.
- 비동기 요청 상태(loading/saving/error)를 명시적으로 관리한다.
- 자동저장 debounce는 기본 1000ms를 사용한다.
- 모든 기능은 화면 상태를 먼저 반영하고 API는 후행 호출한다.

## 에러/응답
- 서버 공통 응답(`success`, `data`, `message`, `errorCode`, `timestamp`)을 기준으로 파싱한다.
- 400/401/404/409 상태는 분기 처리 규칙을 유지한다.
