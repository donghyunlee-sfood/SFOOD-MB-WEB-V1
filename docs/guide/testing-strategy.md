# Testing Strategy (Web)

## 테스트 범위
- 인증: login/me/logout, 401 처리
- 보드: 목록/생성/수정/이동/삭제
- 메모: 목록/생성/내용/위치/크기/z-index/삭제
- 메모타입: 목록 조회
- 자동저장: 1초 지연 + 실패 롤백
- 줌 UI: Ctrl+Wheel + 수동 버튼 + 퍼센트 표시

## 테스트 레벨
- 단위 테스트: 유틸, 상태 업데이트 로직
- 컴포넌트 테스트: 사용자 인터랙션 기반 동작 검증
- 통합 스모크: 주요 사용자 플로우(로그인 -> 보드 -> 메모 저장)
- 필수 시나리오: 보드 이동(드래그 앤 드롭) 성공/실패 처리

## 실행 규칙
- 코드 변경 시 `npm run test` 우선 실행
- 테스트 스크립트 미구성 시 스모크 커맨드로 대체하고 결과를 문서에 기록
- API 연동 이슈는 재현 절차를 문서화한다
- 화면 선반영 -> API 호출 -> 실패 롤백 흐름을 기능별로 검증한다
