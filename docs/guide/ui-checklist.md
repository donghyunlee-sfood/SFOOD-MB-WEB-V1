# UI 설계 체크리스트

## 문서 정보
- 프로젝트: SFOOD-MB-WEB-V1
- 버전: v1.0
- 작성일: 2026-03-03
- 연계 문서: `docs/guide/requirements.md`, `docs/product/prd.md`, `docs/product/api-spec.md`

## 공통 UI 기준
설명:
- 모든 API 호출은 세션 쿠키 기반(`withCredentials`)으로 동작해야 한다.
- 로딩/성공/오류 상태를 사용자에게 명확히 표시해야 한다.
- 모바일/데스크탑 모두 사용 가능한 반응형 레이아웃을 유지해야 한다.
- 모든 기능은 화면 상태를 먼저 반영한 뒤 API를 호출해야 한다.

체크리스트:
- [x] 공통 레이아웃(헤더/LNB/캔버스/하단 툴바) 반응형 기준 확정
- [x] 공통 로딩 컴포넌트(스켈레톤 또는 스피너) 적용
- [x] 공통 에러 표시(토스트 또는 배너) 정책 적용
- [x] 빈 상태(보드 없음, 메모 없음) UI 정의
- [x] 화면 선반영/실패 롤백 공통 패턴 정의

## 1) 로그인 화면 (Auth)
설명:
- 로그인 화면은 Google OAuth 진입점이다.
- 로그인 성공 시 메인 화면으로 이동해야 하며, 이미 세션이 유효하면 로그인 화면을 건너뛰어야 한다.
- 401/인증 실패는 사용자 메시지로 안내한다.

체크리스트:
- [x] 구글 로그인 버튼 배치 및 클릭 인터랙션 정의
- [x] Google Client ID 연결 (`VITE_GOOGLE_CLIENT_ID`)
- [x] 초기 진입 시 `GET /api/v1/auth/me`로 자동 로그인 분기
- [x] 로그인 성공 시 메인 화면 라우팅
- [x] 로그인 실패 메시지/재시도 UX 정의
- [x] 로그아웃 후 로그인 화면 복귀 동작 확인
- [x] 사용자 정보/로그아웃 버튼이 LNB 하단에 표시됨

## 2) 메인 화면 구성 (Main Layout)
설명:
- 메인 화면은 LNB(보드 트리), 보드 캔버스, 하단 메모 타입 툴바로 구성된다.
- 초기 로딩 시 보드/메모타입 데이터를 조회하여 기본 상태를 구성한다.

체크리스트:
- [x] LNB/캔버스/툴바 3영역 레이아웃 설계
- [x] 헤더에는 사용자 정보/로그아웃을 배치하지 않음
- [x] LNB 하단 사용자 정보/로그아웃 섹션 설계
- [x] 최초 진입 로딩 순서 정의(세션 -> 보드 -> 메모타입)
- [x] 선택 보드가 없을 때 안내 화면 정의
- [x] 모바일 축소 레이아웃(패널 접기/펼치기) 정책 정의

## 3) 보드 기능 UI (Board)
설명:
- 보드는 트리 구조로 조회/생성/수정/이동/삭제가 가능해야 한다.
- 보드명은 20자 제한이며 삭제는 논리삭제 API를 호출한다.

체크리스트:
- [x] 보드 트리 렌더링(`GET /api/v1/boards`) 연결
- [x] 최상위 보드 추가 버튼 UI(LNB 내부)
- [x] 하위 보드 추가 액션 UI(LNB 내부)
- [x] 보드명 인라인 수정 UI 및 20자 제한 처리
- [x] 보드 이동(드래그 앤 드롭) 시 피드백 UI
- [x] 보드 삭제 확인 팝업/확인 후 UI 제거
- [x] LNB 리사이즈 핸들 UI
- [x] 보드 이동 테스트 시나리오(필수) 체크

## 4) 메모 기능 UI (Memo)
설명:
- 선택된 보드 기준으로 메모를 조회/생성/수정/삭제한다.
- 메모는 편집 모드와 완료 모드를 가진다.
- 위치, 크기, z-index 변경은 각각 API와 동기화되어야 한다.
- 에디터 툴바는 메모 카드와 분리되어 메모 하단에 표시되어야 한다.

체크리스트:
- [x] 보드 선택 시 메모 목록 조회(`GET /api/v1/boards/{boardId}/memos`)
- [x] 메모 생성 UI(`POST /api/v1/boards/{boardId}/memos`)
- [x] 편집모드/완료모드 전환 UI 정의
- [x] 내용 수정 저장(`PATCH /api/v1/memos/{memoId}/content`) 연결
- [x] 드래그 이동 저장(`PATCH /api/v1/memos/{memoId}/position`) 연결
- [x] 리사이즈 저장(`PATCH /api/v1/memos/{memoId}/size`) 연결
- [x] 컨텍스트 메뉴(z-index 변경) UI + 저장 연결
- [x] 메모 삭제 확인 팝업/삭제 반영 UI
- [x] 메모 겹침 시 z-index 시각적 우선순위 확인
- [x] 에디터 툴바 분리 배치(메모 하단) 확인

## 5) 메모 타입 기능 UI (Memo Types)
설명:
- 메모 타입은 하단 툴바에서 조회/선택되고, 새 메모 생성 시 타입 정보를 전달한다.
- 타입별 기본 스타일(색상/모양)이 즉시 반영되어야 한다.

체크리스트:
- [x] 메모 타입 목록 조회(`GET /api/v1/memo-types`) 연결
- [x] 하단 툴바에 타입별 아이템 렌더링
- [x] 선택 상태(활성 타입) 시각화
- [x] 선택 타입으로 메모 생성 요청 데이터 매핑
- [x] 타입별 기본 스타일(배경/모양) 일관성 확인
- [x] 타입 변경 시 생성되는 메모 스타일 차이 확인

## 6) 줌 UI (Zoom)
설명:
- Ctrl+Wheel 줌 외에 수동 조작 버튼이 필요하다.
- 사용자가 현재 배율을 바로 확인할 수 있도록 `%` 값을 표시한다.

체크리스트:
- [x] Ctrl+Wheel 줌 동작 확인
- [x] 수동 줌 버튼(+/-) 배치
- [x] 현재 줌 퍼센트 표시
- [x] 줌 최소/최대 범위 안내 또는 제한 처리

## 7) 자동저장/오류 복원 UI
설명:
- 메모 편집 완료 시 1초 지연 저장을 적용한다.
- 저장 실패 시 사용자에게 알리고 이전 상태로 롤백한다.

체크리스트:
- [x] blur 후 1초 debounce 저장 적용
- [x] 저장 중 상태(스피너/배지) 표시
- [x] 저장 성공/실패 피드백 메시지 정의
- [x] 실패 시 롤백 동작 확인
- [x] 400/401/404/409 에러 코드별 안내 문구 정의
- [x] 화면 먼저 반영 후 API 통신 순서 확인
