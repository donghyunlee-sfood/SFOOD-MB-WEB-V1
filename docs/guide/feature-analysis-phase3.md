# Memo Web Feature Analysis (Phase 3)

## Goal

- Phase 1/2 기능을 유지하면서 실패 상황에서 사용자 복구 경로를 강화한다.
- 운영/릴리즈 문서를 남겨 실제 실행과 검증 절차를 표준화한다.

## Planned Improvements

1. 오류 처리 UX 강화
- 공통 에러 메시지 정규화(빈 메시지/네트워크 오류 대응).
- 오류 영역에 재시도 버튼을 제공해 마지막 실패 동작을 즉시 재실행 가능하게 한다.
- 진행 중 상태를 시각적으로 표시하여 중복 조작을 줄인다.

2. 롤백 안정성 강화
- 자동 저장 실패 시 기존 롤백 동작을 유지하고 사용자에게 실패 원인을 명확히 전달한다.
- 드래그/리사이즈 저장 실패 시 이전 상태 복구를 명확한 메시지와 함께 보장한다.

3. 테스트 확장
- 에러 메시지 정규화 유틸 테스트 추가.
- 기존 줌/중앙좌표/ID/검증/롤백 테스트 회귀 확인.

4. 운영/릴리즈 문서
- 개발 실행/검증/배포 전 체크 순서를 운영 가이드로 문서화.
- 현재 릴리즈 범위(Phase 1~3)와 제약사항을 릴리즈 노트에 기록.
