## 📖 문서 개요

### 목적

본 문서는 웹 기반 개인 메모 웹앱(Post-it Board)의 개발을 위한 소프트웨어 요구사항을 정의한다.
기능정의서를 기반으로 기능 요구사항, 비기능 요구사항, 제약 사항을 구체적으로 명세하여 개발팀과 이해관계자 간의 공통된 이해를 확립하는 것을 목적으로 한다.

### 범위

본 시스템은 사용자가 구글 계정으로 인증한 후 웹 브라우저에서 Post-it 형태의 메모를 보드(Board)에 자유롭게 배치하고 관리할 수 있는 웹 애플리케이션이다.

- **인증 모듈** — 구글 OAuth 기반 로그인/로그아웃 및 세션 관리
- **보드 모듈** — 보드 CRUD, 트리 구조, 드래그 이동
- **메모 모듈** — 메모 CRUD, 위치/크기/노출순서 관리, 에디터 툴바
- **UI/UX 모듈** — 보드 줌, LNB 리사이즈, 자동 저장

### 용어 정의

| 용어 | 정의 |
| --- | --- |
| 보드 (Board) | 사용자가 메모를 배치하는 작업 공간. 노션의 페이지와 동일한 단위이며 트리 구조로 관리된다. |
| 메모 (Memo) | 보드 위에 배치되는 Post-it 형태의 컴포넌트. 내용, 위치, 크기, 스타일을 포함한다. |
| 메모 타입 | 메모 툴바에서 선택 가능한 3~5가지 모양과 기본 색상이 다른 메모 템플릿. |
| LNB | Left Navigation Bar. 보드 트리 구조를 표시하는 좌측 사이드바. |
| 에디터 툴바 | 메모 편집 모드 진입 시 메모 하단에 나타나는 서식 도구 모음 (색상, 폰트, 크기, 체크박스, 리스트). |
| 완료 모드 | 메모 입력/수정이 완료된 상태. textarea → div 전환, 에디터 툴바 소멸, 자동 저장 트리거. |
| 편집 모드 | 메모를 더블클릭하거나 신규 추가하여 내용을 입력/수정하는 상태. |
| HIDE 처리 | 데이터를 물리 삭제하지 않고 HIDE 컬럼을 true로 설정하여 조회에서 제외하는 논리 삭제 방식. |
| 세션 | 로그인 성공 후 API 서버에 저장되는 인증 정보. 24시간 후 만료. |

## 🖥️ 시스템 개요

### 제품 개요

Post-it Board는 사용자가 웹 브라우저에서 디지털 화이트보드를 활용하는 것처럼 메모를 자유롭게 배치, 편집, 관리할 수 있는 개인용 메모 웹 애플리케이션이다.
별도의 설치 없이 구글 계정으로 로그인하여 바로 사용 가능하며, 모든 데이터는 자동 저장된다.

### 주요 사용자

- **1차 사용자** — 구글 계정을 보유한 개인 사용자
- **액터** — 인증된 사용자 (Google OAuth 완료 후 세션 유지 상태)

### 시스템 컨텍스트

| 레이어 | 역할 |
| --- | --- |
| 클라이언트 (웹 브라우저) | 사용자 인터랙션, UI 렌더링, 자동 ID 생성 |
| API 서버 | 인증 처리, 세션 관리, CRUD API 제공, 비즈니스 로직 |
| 데이터베이스 | 사용자, 보드, 메모 데이터 영구 저장 (논리 삭제 방식) |

### 기능 범위 요약

| ✅ 포함 기능 | ❌ 제외 기능 (자동 생성 금지) |
| --- | --- |

| 보드 조회 / 생성 / 수정 / 삭제 / 이동 | ID/PW 로그인 |
| 메모 조회 / 등록 / 수정 / 삭제 | 사용자 정보 페이지 |

| 에디터 툴바 | 보드 · 메모 정렬 / 검색 |

### 가정 사항

- 사용자는 구글 계정을 보유하고 있다.
- 클라이언트는 웹 브라우저 환경에서만 사용한다. 모바일 앱은 별도 고려하지 않는다.
- 동일 계정의 다중 탭/창 동시 편집에 대한 동기화 처리는 현재 범위에서 제외한다.
- 메모 에디터는 Rich Text Editor 방식으로 구현하되 외부 에디터 라이브러리 사용을 허용한다.
- 보드 줌 기능은 별도 저장 없이 브라우저 세션 동안 유지된다.
- 1초 자동 저장 딜레이는 초기 기본값이며 성능 테스트 후 변경될 수 있다.
