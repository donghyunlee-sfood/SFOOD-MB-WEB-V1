## 인증 (AUTH)
- AUTH-001: 로그인 페이지에서 구글 계정 기반 로그인 요청을 지원한다.
- AUTH-002: 로그인 성공 시 메인 화면으로 이동한다.
- AUTH-003: 세션 유효 시 새로고침/재진입에도 로그인 상태를 유지한다.
- AUTH-004: 세션 만료/무세션 응답(401) 시 로그인 화면으로 이동하고 안내 메시지를 노출한다.
- AUTH-005: 로그아웃 요청 성공 시 세션 상태를 초기화한다.
- AUTH-006: 사용자 정보와 로그아웃 버튼은 헤더가 아닌 LNB 하단 영역에 배치한다.

## 보드 (BOARD)
- BOARD-001: 로그인 후 보드 목록을 조회해 LNB 트리로 렌더링한다.
- BOARD-002: 최상위/하위 보드 생성이 가능해야 한다.
- BOARD-003: 보드명 수정(20자 제한) 기능을 제공한다.
- BOARD-004: 보드 이동(부모/정렬순서) 기능을 제공한다.
- BOARD-005: 보드 삭제는 UI에서 제거되며 서버는 논리삭제를 수행한다.
- BOARD-006: LNB 너비 조절 UI를 제공한다.
- BOARD-007: 최상위 추가/하위 추가 UI는 LNB 영역에서만 제공한다.
- BOARD-008: 보드 이동(드래그 앤 드롭) 테스트는 필수 검증 항목이다.

## 메모 (MEMO)
- MEMO-001: 보드 선택 시 해당 보드 메모 목록을 조회한다.
- MEMO-002: 메모 생성 시 타입/내용/위치/크기/z-index를 포함해 저장한다.
- MEMO-003: 내용 수정 완료(blur) 시 자동저장 트리거를 적용한다.
- MEMO-004: 메모 드래그 이동 후 위치 저장 API를 호출한다.
- MEMO-005: 메모 리사이즈 후 크기 저장 API를 호출한다.
- MEMO-006: 컨텍스트 메뉴 기반 z-index 순서 변경을 지원한다.
- MEMO-007: 메모 삭제 시 UI에서 제거한다.
- MEMO-008: 에디터 툴바는 메모 카드 내부가 아닌 메모 컴포넌트 하단 분리 영역에 표시한다.

## 메모 타입 (MEMO TYPE)
- TYPE-001: 앱 초기 진입 시 활성 메모 타입 목록을 조회한다.
- TYPE-002: 메모 생성 툴바에서 타입별 스타일(색/모양)을 구분해 표시한다.
- TYPE-003: 메모 툴바에서 선택한 타입에 따라 스타일이 다른 메모를 생성한다.

## 줌 (ZOOM)
- ZOOM-001: 보드 줌 사용 시 현재 줌 퍼센트(%)를 화면에 표시한다.
- ZOOM-002: Ctrl+Wheel 외에 수동 줌 버튼(+/- 또는 preset)을 제공한다.

## 자동저장/오류 처리
- AUTO-001: 완료 모드 전환 후 1초 지연 저장을 적용한다.
- AUTO-002: 저장 실패 시 사용자에게 오류를 노출하고 UI 상태를 롤백한다.
- ERR-001: 공통 에러 응답(`errorCode`, `message`)을 사용자 메시지로 매핑한다.
- ERR-002: 400/401/404/409 상태코드를 구분 처리한다.
